# Personal Assistant CLI
Консольний застосунок "Персональний помічник" для збереження контактів.
Проєкт виконаний в рамках домашніх завдань курсу Python Core.

## Завдання 1 — Віртуальне середовище
Для створення та управління віртуальним середовищем використано Poetry.

### Версія Python
Проєкт використовує:
Python 3.12

Версія Python зафіксована у файлі:
pyproject.toml

### Структура проєкту
```
goit-ds-hw-01/
│
├── pyproject.toml
├── poetry.lock
├── README.md
│
└── src/
    └── goit-ds-hw-01/
        └── contact_bot.py
```
Структура src використовується відповідно до рекомендацій Python packaging.

## Встановлення залежностей

### Встановлення середовища:
```
poetry install
```
### Активація середовища:
```
poetry shell
```
або запуск без активації:
```
poetry run contact-bot
```

## Завдання 2 — Docker
Для запуску застосунку в ізольованому середовищі створено Dockerfile.

### Створення Docker образу

У корені проєкту виконайте:
```
docker build -t contact-bot .
```
### Запуск контейнера

Для взаємодії з CLI застосунком контейнер запускається в інтерактивному режимі:
```
docker run -it contact-bot
```
### Збереження даних

Адресна книга зберігається у файл:

addressbook.pkl

Щоб дані не зникали після перезапуску контейнера, використовується Docker volume:
```
docker run -it -v $(pwd)/data:/app/data contact-bot
```
У цьому випадку:

папка data на хості зберігає файл

файл addressbook.pkl створюється всередині неї

дані зберігаються між перезапусками контейнера

Структура після запуску:
```
contact_bot_docker/
│
├── data/
│   └── addressbook.pkl
```
### Запуск команд всередині контейнера

Якщо контейнер вже працює, можна підключитися до нього:

docker exec -it <container-id> /bin/bash
